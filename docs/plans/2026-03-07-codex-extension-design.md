# Codex Extension Design: Skills, Agents, and MCP

**Date**: 2026-03-07
**Status**: Approved
**Author**: Claude Code

## Summary

Create `arckit-codex/` as a standalone Codex CLI extension (following the pattern of `arckit-gemini/` and `arckit-opencode/`) that includes skills, agent configurations, and MCP server entries — features previously only available in the Claude Code plugin.

## Background

As of March 2026, Codex CLI supports:

- **Skills**: Open Agent Skills specification (`.agents/skills/{name}/SKILL.md` + `references/`)
- **Multi-agent**: Experimental, configured via `config.toml` with role definitions
- **MCP servers**: Configured in `config.toml`, auto-launched at session start
- **Hooks**: Not supported (no equivalent to Claude Code's PreToolUse/PostToolUse/SessionStart)

The ArcKit Claude Code plugin has 4 skills, 6 agents, 4 MCP servers, and 14 hooks. This design ports the first three; hooks are not portable.

## Skills Porting

### Compatibility

The Claude Code plugin skills already follow the Open Agent Skills specification (`SKILL.md` + `references/`). Codex CLI scans `.agents/skills/` directories. The structure is 1:1 compatible.

### Changes Required Per Skill

| Skill | Changes Needed |
|-------|----------------|
| `mermaid-syntax` | None — pure reference material with relative paths |
| `plantuml-syntax` | None — pure reference material with relative paths |
| `wardley-mapping` | None — pure reference material with relative paths |
| `architecture-workflow` | Rewrite `/arckit:X` to `/prompts:arckit.X`, remove SessionStart hook reference |

### architecture-workflow Rewrites

- `/arckit:*` command references -> `/prompts:arckit.*`
- "AskUserQuestion" tool -> keep as-is (Codex has interactive prompting)
- "SessionStart hook" reference -> remove or note as unavailable
- `${CLAUDE_PLUGIN_ROOT}` paths -> `.arckit` (same as commands)

### User Installation

```bash
# Per-project (symlink)
mkdir -p .agents
ln -s /path/to/arckit-codex/skills .agents/skills

# Global
cp -r /path/to/arckit-codex/skills/* ~/.agents/skills/
```

## Agent Configuration

### Approach

Each agent gets a separate `.toml` config file referenced from the main `config.toml` via `config_file`. This keeps `config.toml` clean while allowing full agent prompts.

### config.toml Agent Section

```toml
[agents]
max_threads = 3
max_depth = 1
job_max_runtime_seconds = 600

[agents.roles.arckit-research]
description = "Market research, vendor evaluation, build vs buy, TCO analysis. Use when the user needs technology research or vendor comparison."
config_file = "agents/arckit-research.toml"

[agents.roles.arckit-datascout]
description = "Data source discovery, API catalogue search, scoring. Use when the user needs to find data sources or APIs."
config_file = "agents/arckit-datascout.toml"

[agents.roles.arckit-aws-research]
description = "AWS service research via AWS Knowledge MCP. Use when evaluating AWS services."
config_file = "agents/arckit-aws-research.toml"

[agents.roles.arckit-azure-research]
description = "Azure service research via Microsoft Learn MCP. Use when evaluating Azure services."
config_file = "agents/arckit-azure-research.toml"

[agents.roles.arckit-gcp-research]
description = "GCP service research via Google Developer Knowledge MCP. Use when evaluating GCP services."
config_file = "agents/arckit-gcp-research.toml"

[agents.roles.arckit-framework]
description = "Transform artifacts into structured framework with principles, patterns, and guidance."
config_file = "agents/arckit-framework.toml"
```

### Per-Agent .toml Files (Generated)

Each `agents/arckit-{name}.toml` is auto-generated from the plugin agent markdown:

```toml
# Auto-generated from arckit-plugin/agents/arckit-research.md
# Do not edit — edit the source and re-run scripts/converter.py

developer_instructions = """
You are an enterprise architecture market research specialist...
[full agent prompt extracted from markdown, with paths rewritten]
"""
```

### Generation Logic

The converter:

1. Reads `arckit-plugin/agents/arckit-{name}.md`
2. Strips agent-specific YAML frontmatter (`name`, `description`, `model`)
3. Extracts the prompt body
4. Rewrites `${CLAUDE_PLUGIN_ROOT}` paths to `.arckit`
5. Wraps in TOML format as `developer_instructions`
6. Writes to `arckit-codex/agents/arckit-{name}.toml`

## MCP Server Configuration

### config.toml MCP Section

Generated from `arckit-plugin/.mcp.json`:

```toml
[mcp_servers.aws-knowledge]
type = "http"
url = "https://knowledge-mcp.global.api.aws"

[mcp_servers.microsoft-learn]
type = "http"
url = "https://learn.microsoft.com/api/mcp"

[mcp_servers.google-developer-knowledge]
type = "http"
url = "https://developerknowledge.googleapis.com/mcp"
headers = { "X-Goog-Api-Key" = "${GOOGLE_API_KEY}" }

[mcp_servers.datacommons-mcp]
type = "http"
url = "https://api.datacommons.org/mcp"
headers = { "X-API-Key" = "${DATA_COMMONS_API_KEY}" }
```

### Generation Logic

The converter reads `.mcp.json` and outputs TOML `[mcp_servers.*]` sections. HTTP type maps directly. Headers with environment variables use `${VAR}` syntax (same in both formats).

## What Cannot Be Ported (Hooks)

Codex CLI has no hook/event system. The following 14 hooks are **not portable**:

| Hook | Event | Purpose |
|------|-------|---------|
| `arckit-session.mjs` | SessionStart | Initialize session context |
| `arckit-context.mjs` | UserPromptSubmit | Inject project context |
| `secret-detection.mjs` | UserPromptSubmit | Detect secrets in prompts |
| `sync-guides.mjs` | UserPromptSubmit | Sync guides before /pages |
| `health-scan.mjs` | UserPromptSubmit | Pre-scan for /health command |
| `traceability-scan.mjs` | UserPromptSubmit | Pre-scan for /traceability |
| `governance-scan.mjs` | UserPromptSubmit | Pre-scan for /analyze |
| `validate-arc-filename.mjs` | PreToolUse (Write) | Validate ARC-xxx filenames |
| `file-protection.mjs` | PreToolUse (Edit/Write) | Protect critical files |
| `secret-file-scanner.mjs` | PreToolUse (Edit/Write) | Scan files for secrets |
| `update-manifest.mjs` | PostToolUse (Write) | Update docs manifest |
| `allow-mcp-tools.mjs` | PermissionRequest | Auto-allow MCP tools |
| `validate-wardley-math.mjs` | PreToolUse | Validate Wardley map math |

Some hook functionality could be partially replicated via AGENTS.md instructions (e.g., "always validate ARC filenames before writing"), but this is advisory rather than enforced.

## Converter Changes

### New AGENT_CONFIG Entry

```python
"codex_extension": {
    "name": "Codex Extension",
    "output_dir": "arckit-codex/prompts",
    "filename_pattern": "arckit.{name}.md",
    "format": "markdown",
    "path_prefix": ".arckit",
    "extension_dir": "arckit-codex",
    "copy_commands_to_extension": True,
    "copy_agents_to_extension": True,
}
```

### New Functions

1. **`generate_codex_config_toml(mcp_json_path, agents_dir, output_path)`**
   - Reads `.mcp.json` -> generates `[mcp_servers.*]` sections
   - Reads agent files -> generates `[agents.roles.*]` sections with `config_file` references
   - Writes `arckit-codex/config.toml`

2. **`generate_agent_toml_files(agents_dir, output_dir)`**
   - For each agent `.md` file, extracts prompt body
   - Rewrites paths (`${CLAUDE_PLUGIN_ROOT}` -> `.arckit`)
   - Writes `arckit-codex/agents/arckit-{name}.toml`

3. **`rewrite_skill_commands(skills_dir, command_format)`**
   - Walks skill files looking for `/arckit:` references
   - Rewrites to target command format (`/prompts:arckit.`)
   - Removes Claude Code-specific references (hooks, Task tool)

### Existing Function Updates

- `copy_extension_files()`: Already copies skills, templates, scripts, guides. Just needs `arckit-codex` recognized as an extension target.

### Relationship to Existing .codex/ Output

The existing `codex` entry in AGENT_CONFIG continues generating `.codex/prompts/` for `arckit init --ai codex` users. The new `codex_extension` entry generates the standalone extension. Both use the same commands but serve different distribution models.

## User Setup Guide (README.md)

```bash
# 1. Clone the extension
git clone https://github.com/tractorjuice/arckit-codex

# 2. Install skills
mkdir -p ~/.agents/skills
cp -r arckit-codex/skills/* ~/.agents/skills/

# 3. Merge config into your Codex config
# Append MCP and agent sections to ~/.codex/config.toml
cat arckit-codex/config.toml >> ~/.codex/config.toml

# 4. Set CODEX_HOME for commands
export CODEX_HOME="$(pwd)/arckit-codex"

# 5. Start Codex
codex --auto
```

## Feature Parity Summary

| Feature | Claude Code Plugin | Codex Extension | Gap |
|---------|-------------------|-----------------|-----|
| Commands (57) | Plugin auto-discovery | CODEX_HOME prompts | None |
| Skills (4) | Plugin auto-discovery | .agents/skills/ | None |
| Agents (6) | Task tool delegation | config.toml roles | Experimental in Codex |
| MCP (4 servers) | .mcp.json | config.toml | None |
| Hooks (14) | hooks.json events | Not available | Full gap |
| Templates | ${CLAUDE_PLUGIN_ROOT} | .arckit/ | None |
| Scripts | ${CLAUDE_PLUGIN_ROOT} | .arckit/ | None |
| Guides | ${CLAUDE_PLUGIN_ROOT} | .arckit/ (docs/) | None |
